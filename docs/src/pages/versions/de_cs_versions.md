## 版本控制策略

我們認識到您需要來自Material-UI庫的 **穩定性**。 穩定性確保可重用的組件和庫，教程，工具和學習實踐不會意外地過時。 穩定性對於Material-UI蓬勃發展的生態系統至關重要。

本文檔包含 **我們遵循的做法** ，為您提供前沿的UI庫，平衡穩定性。 我們努力確保始終以可預測的方式引入未來的變化。 我們希望每個依賴於Material-UI的人都知道添加新功能的時間和方式，並在刪除過時功能時做好充分準備。

Material-UI嚴格遵循 [Semantic Versioning 2.0.0](https://semver.org/)。 Material-UI版本號有三個部分： `major.minor.patch`。 版本號根據發行版中包含的更改級別遞增。

- **主要版本** 包含重要的新功能，有些但在更新期間預計會有最少的開發人員幫助。 更新到新的主要版本時，您可能需要運行更新腳本，重構代碼，運行其他測試以及學習新API。
- **次要版本** 包含重要的新功能。 次要版本完全向後兼容;更新期間不需要開發人員幫助，但您可以選擇修改應用程序和庫，以開始使用發行版中添加的新API，功能和功能。
- **補丁版本** 風險較低，包含錯誤修復和小的新功能。 更新期間不需要開發人員協助。

## 發布頻率

我們致力於定期發布，以便您可以通過Material-UI的不斷發展來規劃和協調您的更新。

通常，您可以期待以下發布週期：

- 每6個月發布 **主要** 版本。
- 每個主要版本1-3 **次** 次發布。
- 每週發布 **補丁** （任何時候緊急錯誤修復）。

## 支持政策

我們只支持最新版本的Material-UI。 我們還沒有資源提供 [LTS](https://en.wikipedia.org/wiki/Long-term_support) 版本。

## 棄用做法

有時， **“中斷更改”**，例如刪除對選定API和功能的支持，是必要的。

為了使這些轉換盡可能簡單，我們向您做出兩項承諾：

- 我們努力減少重大變更的數量，並儘可能提供遷移工具。
- 我們遵循此處所述的棄用政策，因此您有時間將應用更新為最新的API和最佳做法。

為幫助確保您有足夠的時間和明確的更新路徑，這是我們的棄用政策：

- 我們在更改日誌中公佈已棄用的功能，並在可能的情況下在運行時發出警告。
- 當我們宣布棄用時，我們還會公佈推薦的更新路徑。
- 我們支持在棄用期間現有使用穩定的API，因此您的代碼將在此期間繼續工作。
- 我們僅在主要版本中進行需要更改應用程序的對等依賴項更新（React）。
